# Pesquisa profunda de produto

Roda **antes ou em paralelo** à Análise VYRAL — objetivo: chegar no roteiro já sabendo qual
benefício focar, qual ângulo/público, e o que já funciona para produtos parecidos.

## 1. Receber o produto

O usuário sempre envia a imagem do produto (obrigatório). Use a imagem para identificar:
nome/marca, categoria, formato de uso (tópico, oral, aplicador, etc.).

## 2. Pesquisa nativa (busca na web)

Pesquise reviews reais, páginas de marketplace e vídeos de venda do produto (ou de produtos da
mesma categoria/ativo, se o produto for muito novo/nichado) e monte um mini-dossiê em PT-BR:

```
📋 Descrição — o que é, ativos/ingredientes principais, modo de uso.
🎯 Público-alvo — perfil demográfico + comportamental.
💡 Motivos de compra (3–5).
💭 Dores ocultas — frases/pensamentos íntimos do comprador, com citações reais entre aspas
   quando encontradas em reviews.
⭐ Qualidades destacadas pelos compradores.
```

## 3. Menu de escolha

Feche a pesquisa com um menu de 3 listas para o usuário escolher 1 tripla por roteiro/vídeo:

- **ÂNGULOS** (ex.: exclusividade, choque de preço, transformação, comparação com concorrente)
- **PÚBLICOS** (ex.: perfil demográfico ou dor específica que o produto resolve)
- **BENEFÍCIOS** (o benefício único a focar naquele vídeo específico)

Se o usuário pedir várias variações do mesmo produto, sugira uma tripla diferente por variação,
para que os vídeos não fiquem repetitivos entre si.

## 4. (Opcional) Benchmark externo

Se o usuário já tiver dados de benchmark de TikTok/concorrência prontos (prints, planilhas,
exports de ferramentas como Kalodata), aceite como insumo adicional para refinar o menu da Seção
3 — mas isso não é obrigatório para seguir o pipeline; a pesquisa nativa (Seção 2) já é suficiente
como caminho padrão.

## 5. Saída desta etapa

Dossiê de produto/público + benefício(s) a focar + menu de ângulos/públicos/benefícios. O usuário
aprova ou ajusta antes de seguir para o Briefing.
